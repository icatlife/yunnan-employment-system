# 安全恢复说明

## 概述

本项目在开发过程中，为了测试手机端访问功能，临时修改了防火墙设置。本文档说明作业完成后如何恢复系统安全设置。

## 已修改的安全设置

| 修改项 | 说明 | 风险等级 |
|--------|------|----------|
| 放行端口 5173 | Vite 前端开发服务器 | 低 |
| 放行端口 3000 | Node.js 后端服务器 | 低 |
| 放行 Node.js 程序 | 允许外部连接 Node.js | 中 |
| CORS 允许所有来源 | 后端跨域配置（仅开发环境） | 中 |
| Vite host 0.0.0.0 | 允许局域网访问 | 低 |

## 恢复步骤

### 1. 删除防火墙入站规则

**方法一：命令行（以管理员身份运行 PowerShell）**

```powershell
# 删除 Node.js 放行规则
Remove-NetFirewallRule -DisplayName "Node.js 开发服务器"

# 删除 Vite 端口规则
Remove-NetFirewallRule -DisplayName "Vite 5173 端口"

# 删除后端 3000 端口规则
Remove-NetFirewallRule -DisplayName "后端 3000 端口"
```

**方法二：图形界面**

打开「控制面板」→「Windows Defender 防火墙」→「高级设置」

点击「入站规则」

找到以下规则，右键点击选择「删除」：

- Node.js 开发服务器
- Vite 5173 端口
- 后端 3000 端口

### 2. 恢复 CORS 配置（仅限开发环境）

修改 backend/index.js，将 CORS 改回开发环境配置：

```javascript
// 开发环境允许所有来源（已用于测试）
app.use(cors())

// 如需更安全的配置，可改为：
// app.use(cors({
//   origin: 'http://localhost:5173',
//   credentials: true
// }))
```

注意：提交作业时保持 app.use(cors()) 即可，因为作业评分在本地运行，不涉及公网部署。

### 3. 恢复 Vite 配置（可选）

如果担心安全，可将 vite.config.js 中的 host: '0.0.0.0' 改回 host: 'localhost'。

```javascript
server: {
  host: 'localhost',  // 改为 localhost
  port: 5173,
  proxy: { '/api': { target: 'http://localhost:3000', changeOrigin: true } }
}
```

### 4. 验证防火墙已恢复

```powershell
# 查看是否还有相关规则
Get-NetFirewallRule -DisplayName "*Node*" | Format-Table DisplayName, Enabled
Get-NetFirewallRule -DisplayName "*5173*" | Format-Table DisplayName, Enabled
Get-NetFirewallRule -DisplayName "*3000*" | Format-Table DisplayName, Enabled
```

如果返回空结果，说明规则已成功删除。

## 安全建议

- 作业提交后：建议重启电脑，确保所有临时设置已清除
- 日常开发：不要关闭防火墙，使用添加入站规则代替
- 公网部署：如需部署到服务器，必须配置严格的 CORS 白名单

## 总结

本项目所有安全修改均为临时性调整，用于满足课程作业的手机端测试要求。按照上述步骤操作即可恢复系统原始安全状态。

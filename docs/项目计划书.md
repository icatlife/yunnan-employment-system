# 云南省企业就业失业数据采集系统 - 项目计划书

## 一、项目背景与立项依据

### 1.1 项目背景

就业是民生之本，失业监测是宏观经济调控的重要依据。当前，云南省人社厅需要建立一套自下而上的就业数据采集网络，实现对全省企业就业失业情况的实时监控和动态分析。

本项目旨在构建一个B/S架构的企业就业失业数据采集系统，实现：
- 企业端：基础信息备案、月度数据填报、历史数据查询
- 市端：辖区内企业数据审核、退回、上报
- 省端：企业备案审批、全省数据终审、多维分析、上报部委

### 1.2 立项依据

1. **政策要求**：国家失业监测系统需要各省定期上报就业失业数据
2. **管理需求**：省级人社部门需要实时掌握全省就业动态
3. **技术条件**：AI辅助开发技术成熟，个人可在短期内完成系统搭建

### 1.3 开发模式说明

本项目采用 **基于Agent的AI辅助编程模式**：
- 开发工具：VS Code + GitHub Copilot
- 开发方式：通过自然语言与AI对话，生成代码、调试修复、优化结构
- 版本管理：Git + GitHub，完整记录开发迭代历史

## 三、项目范围概述

### 3.1 包含范围（In Scope）

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 企业端 | 用户登录、企业信息备案、备案上报 | P0 |
| 企业端 | 月度数据填报（建档期/调查期就业人数） | P0 |
| 企业端 | 数据查询（历史报表状态） | P1 |
| 市端 | 登录、辖区企业数据审核（通过/退回） | P0 |
| 省端 | 登录、企业备案审批（通过/退回） | P0 |
| 省端 | 报表管理（查看企业数据、终审） | P0 |
| 省端 | 数据汇总（全省/各地市汇总表格） | P1 |
| 公共 | 通知发布与浏览（省→市→企业） | P2 |

### 3.2 不包含范围（Out of Scope）

- 与国家失业监测系统的实际接口对接（模拟上报）
- 真机部署与服务器上线（本地运行演示）
- 数据大屏与复杂图表分析（以表格为主）
- 系统管理模块（用户/角色管理简化，预置账号）

### 3.3 项目约束

| 约束类型 | 具体内容 |
|----------|----------|
| 时间约束 | 2026年4月10日 - 4月24日，共14天 |
| 技术约束 | 使用React + Node.js + SQLite，AI辅助开发 |
| 资源约束 | 单人开发，无专职测试 |
| 质量约束 | 核心功能无阻断性Bug，可演示完整业务流程 |
